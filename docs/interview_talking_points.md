# Interview Talking Points

## 1. Why I built this
I built TalentPulse because the Talent Acquisition Intern JD asks for Excel, SQL, Power BI/data visualization, automation, AI/scripting, and analytical problem-solving. I wanted one project that proves all of these together in a realistic hiring analytics use case.

## 2. SQL skills
The SQL layer shows joins across candidates, applications, jobs, sources, recruiters, interviews, offers, and stage history. I used CTEs and window functions to calculate funnel conversion, stage drop-off, MoM trends, source quality, recruiter productivity, and time-to-hire.

## 3. Excel skills
The Excel workbook has a formula-driven dashboard, KPI cards, lookup examples, conditional formatting, charts, and monthly summaries. It uses XLOOKUP, COUNTIFS, IFERROR, TEXT, and AVERAGEIF.

## 4. Power BI skills
I designed a Power BI dashboard with pages for executive overview, funnel analysis, source/channel analytics, recruiter productivity, skills intelligence, and diversity/location insights. I also wrote DAX measures for all key business metrics.

## 5. Automation angle
The Python scripts automate database creation, KPI refresh, and weekly report generation. In a real team, this could be scheduled weekly or connected with Power Automate to send reports to recruiters and TA managers.

## 6. AI/scripting angle
The project includes a believable AI-assisted concept: summarizing candidate pipeline health and creating weekly hiring insights. I kept the AI component grounded because the main value is clean data, reliable metrics, and actionable reporting.

## 7. Business impact
This project helps answer questions like: which source is best, where candidates drop off, which recruiter has the strongest pipeline, which department takes longest to hire, and what skills are most common among hired candidates.
